Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

For this project, A client called the Gamining Room, has proposed a game Called "Draw it or Lose it" in which a group of players have to solve puzzles and challenges by drawing, the software requirements of the client were that the game should be able to have one or more teams with each team assigned to it, the game teams must be unique, and that only one instance of game sohuld exist in memory.

What did you do particularly well in developing this documentation?

I think what I did well in the development of this documentation was offering the best reccomendations for what particular operating system the client should ideally be basing the game, I also think I did well in accurately identifying what specific operating systems perform well, their advantages and disadvantages, so that the client can make an accurate determination of how best to identify the challenges and constraints to the software requiremeents.

What about the process of working through a design document did you find helpful when developing the code?

I think the most useful part of the design document that assisted greatly in developing the code was the dissemination of the requirements that need to be met, and the reccomendations of how to overcome the challenges of developing the code. I also think the UML Diagram assist greatly in compartmentalizing and really visualizing the processes of the code and how it works altogether.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

One part of the document that I think should be revised is is the operating system reccomendation, I think due to the Client's software requirements, it is not particularly easy to capture what specific operating system the clients game should be based on, and I think after the evaluation, it leaves the door open for multiple avenues of how to go about creating the game.

Understanding the requirements of the clients is easily known through the comprehensive Software design document, translating the client's requirements into code is easily but understanding the componenets of the software design document, it's evaluation, reccomendations and summary, which all assist in transcribing what the clients needs are into the program. This is a basic neccesity for any developer, so that the developer and his/her client can communicate effectively and share their collective vision of the program.
